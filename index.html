Monadic Russian Roulette

This is a small exploration of the Maybe monad, as implemented in Professor Frisby's Mostly Adequate Guide to Functional Programming. It takes advantage of the monad for conditional React component rendering, preventing a render of an undefined component in a "safe" version of Russian Roulette, and (frequently) crashing in an "unsafe" version of the game.

This was my first introduction to the concept of monads and monadic software design, and I was immediately taken with the concept. It felt like waking up after years of stumbling around in the dark, endlessly bumping my head on unexpected nullish values and implementing dozens of home-rolled undefined and null checks, only to discover that a simple, dependable, and universal solution already existed elsewhere!

Sticky elements with React hooks

A demo of how to use the React hook API to implement a sticky element on a page. Sticky elements look like simple things to implement on their face, but there are a ton of things to consider: scrolling, window resizing, the dimensions of the sticky element and its anchor point, etc. The hooks API gives us a nice interface to separate out the various concerns for a component like this.

Calling React components as functions

When you spend most of your time working within a framework, it's easy to fall into the "mental trap" of the framework itself. For React, one of these mental traps is JSX syntax. It makes components easier to write and use, but it also obscures the basic Javascript functionality that actually turns your components into DOM objects.

This is a demo of how to "render" (in other words: call) a React component as a normal function instead of using JSX. I wouldn't exactly call it a best practice approach, and wouldn't do it in production, but it is a helpful reminder that, at the end of the day, everything we do in React boils down to plain old function calls.

Partial application and currying: React and vanilla JS

The roots of React can be traced directly to some of the foundational functional programming concepts and techniques. Two of the big ones -- partial application and currying -- have all kinds of potential both inside and outside the framework, but it might be hard to see how these two techniques work outside of the React API if your first exposure to them was via React itself.

This demo shows how to do partial application and currying, both with React and without React, to help you draw parallels between them. Hopefully it will help you bridge the conceptual gap so you can start using these techniques in your vanilla JS code!
